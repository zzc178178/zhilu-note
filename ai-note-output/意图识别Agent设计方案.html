为什么需要意图识别

在 CHATBI（智能数据问答）系统中，用户以自然语言输入，AI 需要准确理解用户的真实目的，将输入映射到预定义的意图类别，避免流程跑偏。

6 种意图分类

意图说明示例
QUERY查询类上个月的营收是多少？
HI寒暄类你好、早上好
SUMMARY总结类帮我总结一下前面的分析
UNRELATED无关类今天天气怎么样？
FILE文件类上传新的销售数据
ASK反问类你说的哪个部门？

核心概念：槽位（Slot）

槽位是意图的「参数」，用于从用户输入中提取具体信息。例如：

查询「上个月北区的销售额」→ 意图 QUERY，槽位 {时间=上个月, 区域=北区, 指标=销售额}

槽位类型

  • 必填槽位：缺失时必须追问
  • 可选槽位：有则提取，无则忽略
  • 枚举槽位：预定义值列表
  • 时间槽位：自然语言时间解析
  • 实体槽位：业务实体名称匹配

大模型意图识别流程

执行链路：
用户输入 → LLM 意图匹配 + 槽位提取 → 槽位齐全则执行，缺失则追问 → 返回结构化结果

Prompt 设计要点

  • 明确角色和目标
  • 输出格式严格限定 JSON
  • 提供意图列表 + 槽位定义 + 示例
  • 含历史对话上下文
  • 槽位值缺失时置空，必要时提问

优化方案：向量检索预筛选

当意图数量较多时，先将意图向量化存入向量库，每次先检索最相关的 Top-K 意图再传给 LLM，避免 Prompt 膨胀，提升准确率和响应速度。
